访问数组IO流详解

# 简介

区分 Java 的输入和输出：把自己当成程序， 当你从外边读数据到自己这里就用输入（InputStream/Reader）， 向外边写数据就用输出(OutputStream/Writer)

Java 中将数据的输入输出抽象为流，流是一组有顺序的，单向的，有起点和终点的数据集合，就像水流。按照流中的最小数据单元又分为字节流和字符流。

  • **字节流：**以 8 位（即 1 byte，8 bit）作为一个数据单元，数据流中最小的数据单元是字节。
  • **字符流：**以 16 位（即 1 char，2 byte，16 bit）作为一个数据单元，数据流中最小的数据单元是字符， Java 中的字符是 Unicode 编码，一个字符占用两个字节。

# ByteArrayInputStream(byte[] buf)

public ByteArrayInputStream(byte[] buf)
1

创建一个ByteArrayInputStream ，使其使用buf作为其缓冲区数组。 缓冲区数组不被复制。 的初始值pos是0和的初始值count是长度buf 。

  • 参数

    buf - 输入缓冲区。

代码示例:

@Test
public void test1() {
    String s = "Hello World!";
    // 获取字符串的字节数组
    byte[] b = s.getBytes();
    // ByteArrayInputStream实例化时不抛出异常的原因是没有和外界产生关系
    ByteArrayInputStream bais = new ByteArrayInputStream(b);
}

# ByteArrayInputStream(byte[] buf,int offset,int length)

public ByteArrayInputStream(byte[] buf,int offset,int length)
1

创建ByteArrayInputStream使用buf作为其缓冲器阵列。 的初始值pos是offset和的初始值count是的最小offset+length和buf.length 。 缓冲区数组不被复制。 缓冲区的标记设置为指定的偏移量。

  • 参数

    buf - 输入缓冲区。

    offset - 要读取的第一个字节的缓冲区中的偏移量。

    length - 从缓冲区读取的最大字节数。

代码示例:

@Test
public void test1() {
    String s = "Hello World!";
    // 获取字符串的字节数组
    byte[] b = s.getBytes();
    //起始读取位置
    int off=12;
    //最大读取量
    int len=103;

    log.info("字节数组长度:{}",b.length);
    // ByteArrayInputStream实例化时不抛出异常的原因是没有和外界产生关系
    ByteArrayInputStream bais = new ByteArrayInputStream(b, off, len);
}

# 方法详细信息

# 读取数据
# read
public int read()
1

从该输入流读取下一个数据字节。值字节返回为int ，范围为0至255 。如果没有字节可用，因为流已经到达，则返回值-1 。

这个read方法无法阻止。

  • Specified by:

    read在 InputStream

  • 结果

    数据的下一个字节，如果已经达到流的末尾， -1 。

代码示例:

@Test
public void test1() {
    String s = "Hello World!";
    // 获取字符串的字节数组
    byte[] b = s.getBytes();
    //起始读取位置
    int off = 1;
    //最大读取量
    int len = 15;

    log.info("字节数组长度:{}", b.length);
    // ByteArrayInputStream实例化时不抛出异常的原因是没有和外界产生关系
    ByteArrayInputStream bais = new ByteArrayInputStream(b, off, len);

    int num;
    do {
        num = bais.read();
        
        if (num != -1){
            log.info("输出结果:{}", (char) num);
        }
    } while (num != -1);

# read
public int read(byte[] b,
                int off,
                int len)
1
2
3

从此输入流读取最多len字节的数据到一个字节数组。如果pos等于count ，则返回-1以指示文件结束。否则，读取的字节数为k等于len和count-pos中的较小count-pos 。如果k为正，则字节buf[pos]通过buf[pos+k-1]被复制到b[off]通过b[off+k-1]中所执行的方式System.arraycopy 。值k被添加到pos并返回k 。

这个read方法不能阻止。

  • 重写：

    read在 InputStream

  • 参数

    b - 读取数据的缓冲区。

    off - 目标数组中的起始偏移量 b

    len - 读取的最大字节数。

  • 结果

    读取到缓冲区的总字节数，如果没有更多的数据，因为已经到达流的末尾， -1 。

  • 异常

    NullPointerException - 如果 b是 null 。

    IndexOutOfBoundsException - 如果 off为负数，则 len为负数，或 len大于 b.length - off

  • 另请参见：

    InputStream.read()

代码示例:

@Test
public void test1() {
    String s = "Hello World!";
    // 获取字符串的字节数组
    byte[] b = s.getBytes();
    //起始读取位置
    int off = 2;
    //最大读取量
    int len = 15;

    log.info("字节数组长度:{}", b.length);
    // ByteArrayInputStream实例化时不抛出异常的原因是没有和外界产生关系
    ByteArrayInputStream bais = new ByteArrayInputStream(b, off, len);

    int size = bais.available();
    log.info("返回文件的总字节数:{}", size);
    //生成缓存用于存放读入的字节
    byte[] b1 = new byte[size];

    //把数据放到bype[] 中,并返回读入缓冲区的总字节数
    int read = bais.read(b1, 2, 3);

    log.info("读入缓冲区的总字节数:{}", read);

    //将字节数组转化为字符串
    log.info("结果输出:{}", new String(b1));

}

# 跳过字节数
# skip
public long skip(long n)
1

从此输入流跳过n个字节的输入。 如果达到输入流的结尾，则可能会跳过更少的字节。 要跳过的字节的实际数字为k等于n和count-pos中的较小count-pos 。 值k被添加到pos ，返回k 。

  • 重写：

    skip在 InputStream

  • 参数

    n - 要跳过的字节数。

  • 结果

    实际跳过的字节数。

代码示例:

@Test
public void test1() {
    String s = "Hello World!";
    // 获取字符串的字节数组
    byte[] b = s.getBytes();

    log.info("字节数组长度:{}", b.length);
    // ByteArrayInputStream实例化时不抛出异常的原因是没有和外界产生关系
    ByteArrayInputStream bais = new ByteArrayInputStream(b);

    int size = bais.available();
    log.info("返回文件的总字节数:{}", size);

    //跳过多少个字节
    long skip = bais.skip(3);
    log.info("跳过字节数:{}", skip);

    //生成缓存用于存放读入的字节
    byte[] b1 = new byte[size];


    //起始读取位置
    int off = 1;
    //最大读取量
    int len = 6;
    if ((off + len) <= b1.length && (off + len) >= 0) {
        //把数据放到bype[] 中,并返回读入缓冲区的总字节数
        int read = bais.read(b1, off, len);

        log.info("读入缓冲区的总字节数:{}", read);
        //将字节数组转化为字符串
        log.info("结果输出:{}", new String(b1));
    } else {
        log.info("读取长度越界了.....");
    }
}

public int available()
1

# markSupported
public boolean markSupported()
1

    markSupported在 InputStream

    JDK1.1

# mark
public void mark(int readAheadLimit)
1

设置流中当前标记的位置。当构造时，ByteArrayInputStream对象在默认情况下标记为零。它们可以通过此方法标记在缓冲区内的另一个位置。

如果没有设置标记，则标记的值是传递给构造函数的偏移量（如果没有提供偏移量，则为0）。

注意：这个班的readAheadLimit没有意义。

  • 重写：

    mark在类别 InputStream

  • 参数

    readAheadLimit - 标记位置无效之前可以读取的最大字节数限制。

  • 从以下版本开始：

    JDK1.1

  • 另请参见：

    InputStream.reset()
